Performance & Efficiency

Quarkus makes Java supersonic and subatomic, with fast startup times, low memory footprint, and a small disk footprint. This release continues to improve the Quarkus framework's performance and efficiency.

Hibernate ORM 6.2

Hibernate ORM 6.2 is a major upgrade to the main persistence layer of Quarkus. It brings a lot of improvements and new features including migration to the Jakarta Persistence 3.0 specification, performance improvements to JDBC, and more. Learn more about Hibernate 6.2 in Quarkus.

While there are a lot of great features the move to Hibernate 6.2 introduces some breaking changes. We created a Hibernate ORM migration guide to make the upgrade from 5 to 6 as smooth as possible.

Developer Productivity

One of the founding principles of Quarkus is to bring joy to Java developers by creating a frictionless experience through a combination of tools, libraries, extensions, and more. This release continues that mission by making developers more efficient with new tools and services such as Dev UI, Quarkus CLI, and PACT contract testing.

Dev UI

Quarkus 3 includes a new and improved Dev UI with a new look and feel that is more extensible. You can access the Dev UI at http://localhost:8080/q/dev

Check out the Dev UI playlist on Quarkus YouTube channel to learn more.

Quarkus CLI

The Quarkus command line tool lets you create projects, manage extensions and perform essential build and dev commands.

This release includes additional features like the quarkus deploy command that enables the deployment of Quarkus applications to platforms like Kubernetes, Knative, and OpenShift without requiring changes to the project dependencies or configuration

Pact

Quarkus 3 enables Pact contract-based testing capability which has a significant impact on the development process for Quarkus developers. Quarkus with Pact, allows developers to easily integrate Pact into their testing and deployment workflows without any additional setup. This can help to save time and ensure that developers can focus on writing code.

Watch the Quarkus Insights episode about Pact and Quarkus to learn more.

Kubernetes-Native

Quarkus enables Java developers to create performant, easily deployable, and maintainable applications on Kubernetes. This release includes a number of impactful Kubernetes-native features including new Dev Services for Kubernetes deployments.

Kubernetes Dev Services

It can often be difficult for developers to get access to a Kubernetes cluster for local development. Quarkus 3.0 includes Dev Services for Kubernetes that automatically start and configure a Kubernetes API server in dev mode and when running tests utilizing Docker. Dev Services for Kubernetes implements a service discovery mechanism for your Quarkus applications running in dev mode to share a single cluster. Learn more about enabling Dev Services for Kubernetes

Community & Standards

Quarkus starts and ends with the community which includes a vast ecosystem of 400+ extensions that includes support for popular Java APIs and standards. This release continues that mission with new features and standards support.

Java 11 & 17

Quarkus 3 continues to target the minimal version of Java 11 with Java 17 as the recommended version. We will continue to revise the minimal Java version based on user feedback and the uptake of Java 17 in the ecosystem.

Jakarta EE APIs

Quarkus 3 has updated to the latest Jakarta APIs, which includes a community alignment around the new “jakarta.*” package names. We know this is a big change, so In order to make migrating as smooth as possible, we included an automated update tool to minimize your effort. Learn more about migrating to Jakarta EE API’s.

Eclipse MicroProfile 6

Quarkus 3 is aligned with MicroProfile 6.0 and Jakarta EE Core Profile 10. The Micrometer Metrics extension is recommended over the deprecated SmallRye Metrics implementation of MicroProfile Metrics.

Coming Soon: Long-term Support (LTS)

We often get asked about a Quarkus LTS version. The community asked, so we’ll deliver. Starting with Quarkus 3.2, we will introduce the first Quarkus LTS version. We will share more information as we get closer to releasing Quarkus 3.2.

Migrating to 3.0 is super easy

We want to make the migration to Quarkus 3 seamless for our users. With that in mind, we have created a migration guide as well as tooling to facilitate this process.

The move to Hibernate ORM 6.2 is a major upgrade. While it brings a lot of improvements and new features, it also includes some breaking changes. In order to help migrate Quarkus apps with Hibernate, we have created a Hibernate ORM migration guide to help you through the process.
